# Code style

## Файлы

- Один публичный класс или одна публичная top-level функция - один файл

## Нейминг

- Файлы: PascalCase
- Директории: lowercase
- Имена в коде: camelCase для функций/переменных, PascalCase для типов
- Плоские нейминги: не дублировать контекст пути в имени
  - Пример: `features/profile/.../Fragment.kt` -> файл `Fragment.kt`, без `ProfileFragment.kt`
- `Manager`, `Helper`, `Utils` в именах - моветон. Только если это реально общий util в `shared`

## Константы и магия

- Магические значения запрещены
- Константы выносить в `config/Constants.kt` текущего слайса
- Строки/иконки/картинки: только ресурсы

## Дубли

- Повтор кода выносить в функцию/метод
- Копипаста между файлами/слайсами запрещена

## Комментарии

- Комментарии в коде не писать
- Допускаются комментарии `TODO` с указанием ссылки на блокирующую задачу
