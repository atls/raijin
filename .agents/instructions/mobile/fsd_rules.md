# FSD rules

## База

- Слои фиксированы: `App`, `Screens`, `Widgets`, `Features`, `Entities`, `Shared`
- `Fragments` - легаси. Новое не делать. Старое переносить в `Widgets`/`Features`
- `App` и `Shared` без слайсов. Все остальные слои = набор слайсов
- Слайсы одного слоя не импортят друг друга

## Слои

1. **App**

   - Точка сборки: DI, root nav, глобальные провайдеры
   - Может импортить любые слои
   - В `App` нет бизнес-логики/доменных правил

2. **Screens**

   - Точки входа: экраны/роуты
   - Собирают сценарий из `Widgets`/`Features`
   - Импорты: `Widgets` -> `Features` -> `Entities` -> `Shared`

3. **Widgets**

   - Большие самодостаточные куски функциональности или интерфейса
   - Обычно реализуют целый пользовательский сценарий
   - Могут держать state/store сценария
   - Импорты: `Features` -> `Entities` -> `Shared`

4. **Features**

   - Повторно используемые реализации продуктовых действий (бизнес-ценность)
   - Меньше `Widgets`, используются в разных `Widgets/Screens`
   - Импорты: `Entities` -> `Shared`

5. **Entities**

   - Всё про сущность: модели, правила/юзкейсы, DTO/контракты, мапперы, UI сущности
   - Импорты: только `Shared`

6. **Shared**
   - База: utils, primitives, errors, ui-kit, общая infra
   - Импорты: ни на что выше

## Слайсы

- Группируют код по предметной области/цели слайса
- Слайс должен быть независим от других слайсов своего уровня (минимум сцепленности)
- Внутри слайса запрещены импорты других слайсов своего уровня; только вниз по слоям
- Примеры слайсов: `userOverview`, `sugarChart`

---

## Сегменты

- `ui` — отображение: UI-компоненты, стили, форматтеры дат
- `model` — состояние (state/store), контракты, модели, бизнес-правила
- `api` — функции запросов к бэкенду, типы данных, мапперы
- `lib` — библиотечный код, необходимый другим модулям в слайсе
- `config` — конфигурационные файлы, фиче-флаги

Кастомные сегменты - только если есть жёсткая причина

---

## Правила импортов

- Направленные зависимости (для всех слоёв кроме `App`):  
  `Screens` -> `Widgets` -> `Features` -> `Entities` -> `Shared`
- Исключение: `App` - точка сборки, может зависеть от любых слоёв
- Запрещено: циклы и горизонтальные импорты (между слайсами одного слоя)
