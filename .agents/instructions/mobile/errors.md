# Errors

## База
- Возвраты: через `Result`
- По возможности не использовать `try/catch`, вместо них `runCatching`/`mapCatching`

## Ошибки
- Ошибка = конкретный `Throwable`/тип ошибки с предметным смыслом (например `InvalidEmail`)

## Обработка
- Реакция на ошибки: обновление состояния через store (`setError`/state update)
- Не проглатывать ошибки молча; фиксировать их в состоянии или отдавать выше по контракту
