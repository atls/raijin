# UI (`:ui`)

## Что это

- UI-kit + дизайн-система
- Theme + tokens + компоненты/примитивы + обёртки над стандартными Compose/Material компонентами

## Правила
- Theme брать только из `:ui` (ui/theme)
- Хардкод визуала запрещён: цвета/типографика/spacing/radius только через ui/theme/properties

## Wrapper-first
- Сначала искать компонент в `:ui`
- Стандартный компонент использовать только если аналога в `:ui` нет

## Структура компонента (паттерн)
- Публичный entry: `<Component>.kt`
- Внутренности раскладывать по директориям при росте компонента:
  - `styles/` для appearance/shape/variants
  - `state/` для state и редьюсеров
  - `locals/` для CompositionLocal context
  - `addon/` для ui addons
  - `utils/` для локальных вспомогательных функций

## Стили (как искать и где добавлять)
- `styles/appearance` - цвета/alpha
- `styles/shape` - размеры/радиусы/паддинги/типографика/бордеры
- Рядом с каждым `Appearance`/`Shape` есть `Variants.kt`
- Конкретные стили задаются статическими extension-функциями в `Variants.kt`
- Новый стиль добавлять только через `Variants.kt`, не размазывать по компоненту
- Публичный API компонентов `:ui` должен быть стабильным и минимальным (без лишних параметров)
